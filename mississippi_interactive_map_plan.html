3MA Mississippi Legislative Map

Updated MVP status and production roadmap for a WordPress-hosted interactive map focused on Mississippi counties, state legislative districts, elected officials, and medical cannabis program indicators.

Updated May 13, 2026. Public data should be revalidated before launch.

1. Current Product Direction

The MVP is now a public-facing civic intelligence map for 3MA. It lets users inspect Mississippi counties, MS House districts, and MS Senate districts; see relevant elected officials; view priority medical cannabis votes; and compare county-level MMCP patient, practitioner, dispensary, and license counts.

The current build is intentionally narrower than the original concept. It no longer includes a U.S. House boundary option in the live controls, and it does not yet include judicial districts, internal notes, CRM workflows, or private advocacy data.

2. What Has Already Been Built

Built Interactive Map Shell

  • Leaflet-based map in index.html, styles.css, and app.js.
  • OpenStreetMap basemap restored behind Mississippi layers.
  • Responsive left search/filter panel and right detail panel.
  • 3MA-branded header using the logo from the Media folder.

Built Boundary Layers

  • County boundaries are available as a selectable map mode and remain the base geography.
  • MS House districts from the court-approved May 7, 2025 MARIS layer.
  • MS Senate districts from the court-approved May 7, 2025 MARIS layer.
  • Only one boundary mode is active at a time: Counties, MS House, or MS Senate.

Built Elected Officials

  • House and Senate member data imported from the 3MA Mississippi Legislature Tracksheet.
  • Legislator headshots are displayed when matching files are available.
  • Districts are color-coded by party: Republican red, Democrat navy, Independent white.
  • County selections include a House/Senate dropdown so the user can view one chamber at a time.

Built MMCP Data

  • Patients metric uses only num_patients from 5.13.26 County Summary Dataset.csv.
  • Practitioners and other county program totals are imported from the same county summary file.
  • Active license counts and dispensary records are imported from the MSDH business-search CSV.
  • Patient, practitioner, dispensary, and licensed business tallies display only for county selections.

5. Remaining Work for a Production-Ready WordPress Map

A. Data Quality and Refresh Workflow

  1. Confirm every county name in source CSVs maps cleanly to the county GeoJSON names.
  2. Confirm the legislative tracksheet uses final public-facing scores, grades, and vote values.
  3. Maintain the repeatable npm run import:data workflow for the tracksheet, license export, county summary CSV, and headshot matching.
  4. Add a visible "Last updated" date sourced from the current data exports.
  5. Document that MMCP tally cards are exact county-level values and are not estimated for House or Senate district views.

B. UX and Visual QA

  1. Test desktop, tablet, and mobile layouts inside the target WordPress page template.
  2. Confirm the side panel remains readable and usable on smaller screens.
  3. Review color contrast for party fills, selected boundaries, vote tags, and text.
  4. Confirm city labels do not crowd important district geometry at default zoom.
  5. Add loading and error states for slow networks or missing data files.

C. WordPress Hosting

  1. Decide whether to host the map as a static mini-app embedded by iframe or as a custom WordPress plugin/block.
  2. For fastest launch, deploy static files to a stable static host or WordPress-accessible directory and iframe the page.
  3. Set cache headers carefully so data updates can be published without users seeing stale files for too long.
  4. Confirm Content Security Policy compatibility for Leaflet, OpenStreetMap tiles, Nominatim address search, local images, and JSON data files.
  5. Compress GeoJSON and image assets for production load time.

D. Accessibility and Public Readiness

  1. Continue improving keyboard-friendly focus handling for search results, boundary controls, and map selections.
  2. Review alternative text and fallback behavior for missing legislator headshots.
  3. Add clear source notes explaining that MMCP tally cards are county-level values shown only in Counties mode.
  4. Confirm that no private member notes, internal relationship notes, or non-public contact data are included in the public build.

6. Deferred Features

Feature Status Notes
U.S. House boundaries and members Deferred The data file may remain available locally, but the live MVP no longer exposes this boundary option.
Judicial districts Deferred Requires sourcing or building district geometries from county lists or official GIS layers.
Internal CRM/admin mode Deferred Would require authentication, private data handling, and a separate public/private data model.
Address-level access analysis Deferred Current public data supports county-level calculations for most license types. Address-level analysis should be limited to sources that provide addresses.
Automated data dashboard Deferred Future phase could include a backend import process, admin review queue, and scheduled data refresh.

7. Recommended Launch Path

  1. Freeze the current MVP feature set: Counties, MS House, MS Senate, search, official cards, priority votes, and MMCP county metrics.
  2. Run a data audit against the tracksheet, county summary dataset, license export, and headshot directory.
  3. Perform visual QA in the actual WordPress page layout on desktop and mobile.
  4. Deploy the static app to a staging URL and embed it in a private WordPress test page.
  5. Collect internal feedback from 3MA, then make only launch-critical revisions before public release.

Practical recommendation: launch the first public version as a static embedded map. Move to a custom WordPress plugin or authenticated admin version only after the public data model and update workflow are stable.
